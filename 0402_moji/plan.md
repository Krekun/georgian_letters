# 📝 ジョージア文字学習アプリ — 要件定義書（MVP）

**プロジェクト名:** Script Trainer（仮）
**作成日:** 2026-04-01
**ステータス:** MVP要件確定

---

## 1. 背景・課題

海外旅行中に旅先の文字（スクリプト）を学びたいニーズがある。言語学習アプリは多いが、**文字そのものの「読み書き」に特化した**軽量ツールはほぼ存在しない。特にジョージア文字やアルメニア文字のようなマイナー文字体系は、既存アプリのカバー外である。

## 2. プロダクト概要

スマホブラウザで動く、ジョージア文字（მხედრული / ムヘドルリ）の手書き学習Webアプリ。ローマ字表記を見て → なぞり練習 → 自力で書いて比較、という3ステップで文字を覚える。

## 3. ターゲットユーザー

- ジョージア旅行中 or 旅行予定の旅行者
- ジョージア語の前知識ゼロ
- スマホメインで使用

## 4. 対応文字体系（MVP）

| 文字体系 | 文字数 | 対応 |
|---------|--------|------|
| ジョージア文字（ムヘドルリ） | 33文字 | ✅ MVP |
| アルメニア文字 | 39文字 | 🔜 将来対応 |

## 5. 画面構成

### 5.1 ホーム画面
- 文字体系の選択（MVPではジョージアのみ）
- レッスン一覧（進捗表示付き）
- 一覧表への導線

### 5.2 一覧表画面（チャート）
- 全33文字を一覧表示
- 各文字：ジョージア文字 / ローマ字読み / 音声の種類（母音/子音）
- 参照用。タップで拡大表示があると望ましい

### 5.3 おまけ：イントロ画面（レッスン0）
- 「この文字が読めるようになります！」という導入
- 旅行で使う単語・フレーズをジョージア文字 + ローマ字 + 日本語訳で表示
- 読むだけ（書き取りなし）
- 学習のモチベーション付けが目的

### 5.4 レッスン画面（コア体験）

1レッスン = 3〜5文字

#### Step 1: なぞり練習（トレースモード）
- お手本の文字が薄く表示される
- その上を指でなぞる（Canvas描画）
- ローマ字読みと文字名を上部に表示
- 1文字ずつ順番に進む
- 「次へ」ボタンで次の文字に移行

#### Step 2: 自力書き取り（フリーモード）
- ローマ字読みのみ表示（お手本なし）
- 空白のCanvasに自力で書く
- 書き終わったら「答え合わせ」ボタン
- お手本と自分の文字を並べて表示
- 自己申告で「できた ✅ / もう一度 🔄」を選択

#### Step 3: レッスン完了
- そのレッスンの正答率（自己申告ベース）を表示
- 「もう一度」or「次のレッスンへ」を選択

### 5.5 おまけ：フィナーレ画面（レッスン9）
- 全文字を覚えた後の到達確認
- イントロと同じ単語・フレーズを再表示
- 「もう読めるようになったはず！」という達成感の演出
- 読むだけ（書き取りなし）

### 5.6 進捗画面（オプション）
- レッスンごとの完了状態
- 全体の進捗率

## 6. 学習フロー

```
イントロ（旅行単語を紹介）→ レッスン1〜7 → 総復習 → フィナーレ（旅行単語を読む）

各レッスン内:
  なぞり練習（3〜5文字）→ 自力書き取り（同じ文字）→ 結果表示 → ホームに戻る
```

## 7. データ設計

### 7.1 文字データ（ハードコード）

```javascript
{
  character: "ა",     // ジョージア文字
  romanization: "a",  // ローマ字表記
  type: "vowel",      // vowel | consonant
  lesson: 1,          // 所属レッスン番号
  order: 1            // レッスン内の順番
}
```

## 8. 技術要件

| 項目 | 選定 |
|------|------|
| フレームワーク | React（JSX単一ファイル） |
| 描画 | HTML5 Canvas API |
| フォント | Google Fonts — Noto Sans Georgian |
| 状態管理 | useState / useReducer |
| 永続化 | localStorage |
| デプロイ | Claude Artifact / GitHub Pages |
| オフライン | 非対応（MVP） |

### 8.1 Canvas仕様
- タッチイベント対応（`touchstart`, `touchmove`, `touchend`）
- ペンの太さ：3〜4px（スマホ指書き想定）
- 色：黒（書き込み）/ グレー薄め（お手本なぞり用）
- クリアボタン（書き直し用）
- レスポンシブ（画面幅に合わせてCanvas幅を調整）

### 8.2 フォント表示
- お手本文字は大きく表示（80〜120px程度）
- なぞりモードでは `opacity: 0.15` 程度で薄く表示
- Noto Sans Georgian を Google Fonts から読み込み

## 10. スコープ外（MVP）

- アルメニア文字対応
- 手書き認識による自動正誤判定
- 音声読み上げ
- ユーザー登録・クラウド同期
- オフライン対応（Service Worker）
- 単語・文レベルの学習
- スペースドリピティション（SRS）

## 11. 将来の拡張候補

1. **アルメニア文字対応** — 文字データ追加で対応可能な設計にしておく
2. **SRS（間隔反復）** — 苦手文字の復習頻度を自動調整
3. **音声再生** — Web Speech API or 音声ファイル
4. **単語モード** — 文字を覚えた後、簡単な単語の読み書き
5. **PWA化** — Service Worker でオフライン対応
6. **他の文字体系** — タイ文字、ミャンマー文字、アムハラ文字など
